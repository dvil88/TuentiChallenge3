Challenge 18 - Energy will be infinities

« Prev Next »

You are a QA engineer in a famous game development company. Your latest hit is about to be released and everything needs to be perfect. In this latest game, the player's avatar has an specific amount of energy and it can travel among different locations. Usually, it takes energy to travel from location 0 to location 1, but there are "comfortable" paths that in fact increase energy.

Your task is to ensure that maps are bug-free, i.e, that it is not possible for a player to increase her energy infinitely.

Input and limits

  • First line of the input contains the number of test cases, N
    1 ≤ N ≤ 10
  • Number of different locations (V) follows
    1 ≤ V ≤ 50000, locations named 0,..,V-1
  • Next, number of possible movements, E
    1 ≤ E ≤ 50000
  • Next, energy variation for each movement in %
    origin destination energy_variation (-20 ≤ energy_variation ≤ 20)

Output

For each test case, output True if the map is buggy, False otherwise

Sample input

2
5
5
0 1 20
1 2 -20
2 0 1
3 4 20
4 3 -15
5
5
0 1 20
1 2 -20
2 0 1
3 4 10
4 3 -10

Sample output

True
False
